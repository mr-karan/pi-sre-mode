# 005 — MVP

## Objective

Build the smallest useful public package that proves the workflow and overlay model.

## MVP scope

### Public content
- generic SRE methodology skill
- generic incident templates
- one report prompt / report helper flow

### Public extension
- `/incident`
- `/check-connectors`
- `/report`
- `/incident-reset`
- read-only guardrails via `tool_call`
- footer status line
- lightweight persisted incident state
- overlay registration and merge support

### Private overlay compatibility
The MVP must support both:
- zero-code private overlays (skills + prompts only)
- enhanced private overlays (small extra extension)

## MVP command definitions

### `/incident`
Requirements:
- choose template
- enter service
- enter time window
- optionally add notes
- persist result into session state
- update status line

### `/check-connectors`
Requirements:
- run built-in and overlay-defined checks
- show pass/fail/unknown status
- do not mutate anything

### `/report`
Requirements:
- generate markdown using incident state + current context
- either write to file or place in editor
- support overlay report path defaults later

### `/incident-reset`
Requirements:
- clear stored incident state
- reset status line

## Guardrails

v1 should block obvious destructive commands. It does not need a perfect parser.

Minimum blocked patterns:
- `sudo`
- `rm `
- `mv `
- `chmod `
- `chown `
- `kill ` / `pkill ` / `killall `
- `systemctl restart` / `systemctl stop`
- `nomad job stop`
- `nomad job run`
- `nomad alloc stop`
- mutating `aws` create/update/delete/terminate APIs

## State model

Persist incident state in Pi session entries.

Minimum fields:
- mode enabled
- template ID
- service
- since
- notes

## Acceptance criteria

### Public package acceptance
- user can install the package in Pi
- `/incident` configures an investigation context
- next prompt receives the incident-mode instructions automatically
- `/check-connectors` runs shell-based checks
- unsafe commands are blocked before execution
- `/incident-reset` clears the mode cleanly

### Overlay acceptance
- a private package can add skills without changing the public package
- a private package can register templates/checks through a tiny extension
- overlay precedence works predictably

## Explicit deferrals

Do not build these in MVP:
- custom dashboard UI with `ctx.ui.custom()`
- remote llmduck bridge
- typed connector execution framework
- advanced evidence capture model
- automatic RCA generation to org-specific paths
- auto-detection of every possible private setup
